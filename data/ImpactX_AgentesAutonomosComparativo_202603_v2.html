🎯 Como usar OpenClaw para alcançar Viktor-level

1. Heartbeat System (padrão Viktor)

Crie um agente "observer" que roda a cada 30 min:

# Pseudocódigo para agents/observer.md
NAME: observer
SCHEDULE: "*/30 * * * *"  # a cada 30 min

TOOLS:
  - slack.channels.list
  - slack.messages.search
  - gemini.generate  # análise de padrões

INSTRUCTIONS:
  1. List todos canais que você monitora
  2. Para cada canal, busque últimas 50 mensagens
  3. Análise com Gemini: padrões, anomalias, oportunidades
  4. Se encontrar padrão acionável:
     - Proposta de automação em thread
     - Aguarda aprovação de Rafael
     - Cria nova automação se aprovado

2. Multi-Agent Routing (padrão CrewAI melhorado)

# agents/coordinator.md
NAME: coordinator
ROLE: Distribuidor de tarefas inteligente

TOOLS:
  - slack.reactions.add
  - agent.invoke  # chamar outros agentes

INSTRUCTIONS:
  1. Quando receber mensagem complexa:
     - Se contém "code" → invoke code-specialist
     - Se contém "data/analysis" → invoke data-specialist
     - Else → handle direto
  2. Coleta resultado
  3. Responde no Slack com summary + link

3. Rastreamento de Execução (observabilidade)

# scripts/log-run.sh
#!/bin/bash
TIMESTAMP=$(date -u +"%Y-%m-%d %H:%M:%S UTC")
AGENT=$1
INPUT=$2
OUTPUT=$3
DURATION=$4
STATUS=$5  # "success" or "error"

echo "- [$TIMESTAMP] **$AGENT** ($STATUS) | Duration: ${DURATION}s | Input: \`$INPUT\` | Output: \`${OUTPUT:0:50}...\`" >> workspace/execution-log.md

5Personalidade do Bot
🎭
Nome: O bot não tem nome. Se perguntado, responde: "Sou seu assistente. Sem nome." — evita antropomorfização que cria dependência.
✅ É assim que o bot fala
  • Direto e literal — "Humor de 1 a 5?" não "Como você está?"
  • Curto — máx 4 linhas por mensagem, sempre
  • Uma coisa por vez — nunca lista de perguntas
  • Propositivo — sempre termina com 1 ação ou 1 pergunta
  • Factual nas celebrações — "5 dias seguidos. Teu cérebro gravou isso."
  • Neutro no tom — nem frio nem excessivamente caloroso
  • Consistente — mesma estrutura de resposta sempre
🚫 O bot nunca faz isso
  • Metáforas ou ironia
  • "Como você está se sentindo hoje?"
  • Listas de opções ou sugestões múltiplas
  • Pressionar ou cobrar respostas
  • Mais de 1 lembrete por check-in
  • Julgar ou comentar comportamento
  • Diagnóstico ou conselho médico
  • Fingir ser humano quando perguntado
  • Variar o padrão estrutural por humor do usuário
🔑
Princípio central: O bot é previsível por design. Rafael tem TEA — previsibilidade é segurança. Cada interação deve ter o mesmo padrão estrutural independente do conteúdo.
6Skills do Agente — OpenClaw @tools
SKILL 01checkin_tool — Registro de Hábitos

Salva e recupera dados dos check-ins diários. Chamada automaticamente pelo scheduler e quando Rafael responde a um check-in.

@tool def checkin_tool(data: CheckinInput) -> CheckinResult: """Salva check-in diário. Campos: date, medication_taken, sleep_quality(1-5), exercised, nutrition_ok, mood(1-5), top_task_done, had_social_contact, notes. Upsert no Supabase health_checkins (UNIQUE date). Retorna confirmação + streak atual se relevante."""
  • → Input: dict com campos do check-in (aceita parcial)
  • → Output: confirmação + streak do hábito registrado
  • → Trigger: scheduler (automático) + resposta do usuário
SKILL 02journal_tool — Diário Emocional

Guia o processo de expressão guiada. Ativada por palavras-chave. Gera resumo e salva entrada no diário com tags automáticas.

@tool def journal_tool(action: Literal["start","save","read"], text: str = "") -> str: """Gerencia diário emocional. start: inicia sessão de expressão guiada (1 pergunta por vez) save: salva entrada com resumo gerado por LLM read: recupera últimas N entradas Trigger keywords: preciso falar, tô travado, tô mal, /expressar, não tô bem, precisando falar"""
  • → Fluxo: 1 pergunta → escuta → 1 pergunta → resumo + 1 ação
  • → Salva: raw_text + summary + action_suggested + emotion_tags
  • → Nunca mais de 1 pergunta por mensagem no fluxo
SKILL 03pattern_tool — Detecção de Padrões

Roda em loop async a cada 1h. Verifica padrões negativos nos últimos check-ins e dispara ações quando detectados.

@tool def pattern_tool(check_type: Literal["all","mood","exercise","medication"]) -> PatternResult: """PADRÕES MONITORADOS: mood < 2 por 2+ dias consecutivos → severity: CRITICAL → crisis_tool sem exercício por 3+ dias → severity: MEDIUM → motivation_tool sem remédio por 2+ dias → severity: HIGH → motivation_tool streak 5+ dias de hábito → severity: CELEBRATE → motivation_tool"""
  • → Roda: async loop a cada 60 minutos
  • → Salva: pattern_events com severity e timestamp
  • → Dispatcha: motivation_tool ou crisis_tool conforme severity
SKILL 04motivation_tool — Nudges e ACT

Entrega nudges baseados em neurociência comportamental e exercícios de desfusão cognitiva (ACT). Nunca mais de 1 por dia.

@tool def motivation_tool(pattern_type: str, severity: str) -> str: """Seleciona nudge ou exercício ACT adequado ao padrão. Banco: 15 exercícios em act_exercises (seed: data/act_exercises.json) Nunca usar mesmo exercício 2x na mesma semana. Máx 1 nudge proativo por dia total. Celebração (severity=CELEBRATE): mensagem factual curta. Ex: "5 dias seguidos de exercício. Teu cérebro gravou isso." """
  • → Input: tipo de padrão + severidade
  • → Output: mensagem de nudge ≤4 linhas
  • → Restrição: max 1 nudge proativo por dia
SKILL 05 · CRÍTICAcrisis_tool — Protocolo de Crise

Ativada quando mood <2 por 2 dias OU keywords de ideação detectados. Script FIXO — o LLM NÃO improvisa aqui. Hardcoded na tool, não passa pelo AgentExecutor.

@tool def crisis_tool(trigger: Literal["low_mood","ideation_keywords","manual"]) -> str: """PROTOCOLO FIXO — NÃO ALTERAR: 1. "Estou percebendo que pode estar difícil agora." 2. "Você está bem?" 3. Se negativo: fornece contatos + pergunta quem ligar - CVV: 188 (24h) - Psicólogo Rodrigo: (21) 97915-6525 - "Qual nome vem primeiro na sua cabeça?" 4. NÃO tenta resolver sozinho 5. Registra em pattern_events severity=CRITICAL KEYWORDS TRIGGER: "não quero mais", "cansado de tudo", "não adianta", "desaparecer", "sumir", "acabar"""
  • → Script é hardcoded — output não passa pelo LLM
  • → Sempre registra em pattern_events + structured log
  • → test_crisis.py deve cobrir todos os triggers
SKILL 06summary_tool — Relatório Semanal

Roda todo domingo às 18h. Agrega dados da semana, calcula métricas, Claude gera 1 insight de correlação.

@tool def summary_tool(week_start: date, week_end: date) -> WeeklyReport: """Relatório semanal: - % adesão por hábito (remédio, exercício, sono, tarefa, conexão) - humor médio da semana vs semana anterior - 1 correlação identificada pelo LLM (só uma) - streaks records - celebração se algum hábito >= 80% Métricas calculadas por código, não pelo LLM. LLM só escreve o insight de correlação."""
  • → Formato: ≤12 linhas no Telegram
  • → 1 insight de correlação — não lista
  • → Trigger: domingo 18h (scheduler) ou /semana

8Schema do Banco (Supabase)
-- === migrations.py — rodar na primeira vez === -- pgvector para RAG CREATE EXTENSION IF NOT EXISTS vector; -- Check-ins diários (coração do sistema) CREATE TABLE health_checkins ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), date DATE NOT NULL UNIQUE, -- 1 registro/dia, upsert medication_taken BOOLEAN, sleep_quality INTEGER CHECK (sleep_quality BETWEEN 1 AND 5), exercised BOOLEAN, nutrition_ok BOOLEAN, mood INTEGER CHECK (mood BETWEEN 1 AND 5), top_task_done BOOLEAN, had_social_contact BOOLEAN, notes TEXT, created_at TIMESTAMPTZ DEFAULT NOW() ); -- Diário emocional CREATE TABLE emotional_journal ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), date DATE NOT NULL, raw_text TEXT NOT NULL, summary TEXT, action_suggested TEXT, emotion_tags TEXT[], created_at TIMESTAMPTZ DEFAULT NOW() ); -- Histórico de conversas (memória do agente) CREATE TABLE bot_conversations ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), session_id TEXT NOT NULL, role TEXT CHECK (role IN ('user', 'assistant')), content TEXT NOT NULL, created_at TIMESTAMPTZ DEFAULT NOW() ); -- Exercícios ACT CREATE TABLE act_exercises ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), title TEXT NOT NULL, content TEXT NOT NULL, trigger_pattern TEXT, -- 'no_exercise' | 'low_mood' | 'no_meds' | 'general' last_used DATE ); -- Eventos de padrão detectados CREATE TABLE pattern_events ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), pattern_type TEXT NOT NULL, severity TEXT NOT NULL, -- 'CRITICAL' | 'HIGH' | 'MEDIUM' | 'CELEBRATE' action_taken TEXT, resolved BOOLEAN DEFAULT FALSE, detected_at TIMESTAMPTZ DEFAULT NOW() ); -- RAG — chunks dos laudos com embeddings CREATE TABLE document_chunks ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), source TEXT NOT NULL, content TEXT NOT NULL, embedding vector(1536), created_at TIMESTAMPTZ DEFAULT NOW() ); CREATE INDEX ON document_chunks USING ivfflat (embedding vector_cosine_ops); -- Apple Watch v2 (criar após spike de viabilidade) CREATE TABLE apple_health_data ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), date DATE NOT NULL UNIQUE, sleep_duration_min INTEGER, hrv_avg DECIMAL, steps INTEGER, resting_heart_rate INTEGER, synced_at TIMESTAMPTZ DEFAULT NOW() );

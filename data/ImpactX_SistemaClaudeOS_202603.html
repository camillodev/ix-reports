2. Diagnóstico — O que funciona, o que falta, o que sobra

✅
O que está BOM
Stack de MCPs é excelente — 12 conexões cobrindo email, calendário, projeto, código, infra, banco, deploy. Pouquíssimas consultorias do teu tamanho têm isso.
Skills de output são fortes — report HTML, pptx, deploy automático, Linear tickets. A "última milha" (gerar entregável) funciona.
Brand voice documentada — contexto-impact-x é ouro. Toda skill pode herdar a voz IX automaticamente.
Meta-ferramentas prontas — skill-creator + mcp-builder = podes criar qualquer skill nova sem ajuda externa.
🚨
Problema central: Skills funcionam ISOLADAS
Hoje cada skill é uma ilha. Tu usas startup-validator num dia, marketing:content-creation noutro, mas o output de uma não alimenta a outra. O resultado: retrabalho manual de contexto a cada sessão. O ROI por hora de uso do Claude fica limitado porque tu és o "middleware humano" entre as skills.
⚠️
Gaps identificados
1. Zero Scheduled Tasks — Tudo é reativo. Claude só trabalha quando tu pedes.
2. Projeto "Automacoes" vazio — Criado mas sem conteúdo. Sem docs, sem files.
3. Sem skill de prospecção/outreach IX — sales:draft-outreach é genérica, não tem a voz IX.
4. Sem pipeline encadeado — pesquisa → posicionamento → copy → campanha não existe como fluxo.
5. brand-guidelines é da Anthropic — Não da IX. Irrelevante e potencialmente confusa.
ℹ️
Skills subutilizadas que têm MUITO potencial
marketing:* — 7 skills de marketing instaladas via plugin. Nenhuma configurada pra IX.
sales:* — 6 skills de vendas. account-research + call-prep são perfeitas pra G2I.
product-management:* — feature-spec + roadmap. Útil pra entregar valor a clientes.
enterprise-search:* — Busca cross-source (Slack + Notion + Gmail). Não está sendo usada.

7. Conceito-Chave: Por que encadear skills muda tudo

O modelo mental

Pensa assim: cada skill isolada é uma ferramenta. Um martelo. Uma serra. Útil, mas tu precisas pegar, usar, guardar, pegar outra. Skills encadeadas são uma linha de montagem. O material entra numa ponta e sai produto na outra.

Skills isoladas vs. Sistema

AspectoSkills isoladas (hoje)Skills encadeadas (proposta)
ContextoTu repetes contexto toda sessãoContexto flui entre skills automaticamente
Output1 artefato por vezPipeline completo: pesquisa → entregável final
ConsistênciaDepende de tu lembrar do brand voiceBrand voice aplicado automaticamente em todo output
ROI/hora~2x produtividade~5-10x produtividade (menos retrabalho)
EscalabilidadeLimitado ao teu tempo de promptScheduled tasks trabalham mesmo sem tu pedir
✅
A fórmula
Pesquisa (dados reais) → Posicionamento (brand voice IX) → Copy (content-creation) → Campanha (campaign-plan) → Entregável (report/pptx/post)

Cada etapa herda o output da anterior. Tu intervéns só nos pontos de decisão. O resto é sistema.

Pra não esquecer (conceito SSG/ISR/SSR adaptado)

Tu mencionaste que esquece conceitos básicos. O sistema de skills encadeadas funciona igual renderização:

ConceitoNo Next.jsNo Claude OS
SSG (Static)Gera HTML no build timeScheduled Tasks — briefing gerado automaticamente toda manhã, pronto quando tu acorda
ISR (Incremental)Revalida após X tempoPipeline Pulse — atualiza estado dos deals semanalmente, revalida o que mudou
SSR (Server)Gera no requestSkills on-demand — "prep G2I" e Claude gera tudo na hora, sob demanda
